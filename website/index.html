Quick start

Local & private by default - no code leaves your machine until you choose a remote provider and consent.

  1. Set it once. Say it your way. Proveedor listo. Ahora, en español. Choose a model with Spanish support, then describe the change naturally.
    1

    Choose a provider

    human-to-code runs on any platform supported by Node.js 24+. Start local with Ollama, or choose OpenAI when you are ready to explicitly consent to remote generation.

    Provider
  2. Your language. Your workflow. اردو میں لکھیں Pull a multilingual model, then prompt naturally in Urdu when supported.
    2

    Install Ollama & pull the model

    The default config targets local Ollama on http://localhost:11434 with qwen2.5-coder:7b. Install Ollama for your platform, then pull the model once.

    ollama pull qwen2.5-coder:7b
  3. Your language. Your workflow. اردو میں لکھیں Pull a multilingual model, then prompt naturally in Urdu when supported.
    2

    Set your OpenAI key

    Credentials are environment-only - the config stores the variable name (OPENAI_API_KEY), never the key itself.

    export OPENAI_API_KEY="your_api_key"
  4. 3

    Create the config

    --init writes a complete, schema-versioned human-to-code.config.json and never overwrites an existing file.

    npx human-to-code --init .

    The generated config already targets local Ollama - no edits needed.

    Change the provider to OpenAI, add reviewed pricing bounds, and set remoteProviderConsent to true before generating.

  5. 4

    Write & generate

    Create a .human request, then run the command. It shows a receipt and asks before writing.

    npx human-to-code .

Examples

Two direct ways to turn intent into code - pick the one that fits your workflow.

Write your request in a *.human file; the generated code lands in a sibling file named after it (extension follows your configured language).

greet.human Human 
# greet.humanWrite a function that takes a name and returnsa friendly greeting, with a docstring.
Terminalzsh
$ npx human-to-code .yes wrote greet.py
human-to-codeUTF-8   LF   Ready

Drop a marker inside an existing source file using its native block syntax - for example, /* @human ... */ for a multiline TypeScript or JavaScript request and """ @human ... """ in Python. The marker region is replaced in place with generated code.

utils.ts TypeScript 
// utils.ts// @human debounce helper with cancel()
Terminalzsh
$ npx human-to-code .yes wrote utils.ts
TypeScriptUTF-8   LF   Ready

Every generation shows a receipt - files, provider, model - and asks before writing. Nothing is written on --dry-run.

Docs

The essentials - the full README covers everything.

01

Quick start

# default: discover .human files and @human
# markers, show a receipt, write code
npx human-to-code .

# create a reviewable config (never overwrites)
npx human-to-code --init .

Requires Node.js 24+. The default provider is loopback-local Ollama with qwen2.5-coder:7b - no code leaves your machine until you configure a remote provider and consent.

02

Providers

Ollama OpenAI

OpenAI and Ollama (local, Cloud, or a trusted custom endpoint) are the supported providers. Credentials are environment-only - the config stores a variable name like OPENAI_API_KEY, never a key. Remote generation requires explicit consent and reviewed pricing bounds.

03

Commands

  • human-to-code . - direct .human/@human generation with receipt
  • --init - create a reviewable config without overwriting one
  • --dry-run - inspect the work without generating code
  • --yes - accept the receipt without an interactive prompt
04

Safety model

The converter scans model-bound context for secrets, checks generated candidates, and uses guarded writes. Read SECURITY.md.
